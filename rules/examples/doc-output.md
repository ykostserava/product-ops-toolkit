# Documentation Output Standards

Example of a **`paths:`-scoped rule**: loads only when Claude works on files under
`docs/`, so it costs zero context in coding sessions. Edit to match your templates.

## Product specifications (PRD, user stories)
- Follow your PRD template (see `templates/prd-template.md` in this toolkit)
- Every user story has acceptance criteria (Given-When-Then format)
- Link to the tracker epic/parent issue
- Include measurable success metrics
- Be specific, testable, actionable

## Documentation
- Structured, stakeholder-ready
- Use templates when available
- Cross-link to tracker issues
- Keep updated as single source of truth

## Reports & analysis
- Data-driven insights
- Actionable recommendations
- Use tables for clarity
- Focus on decisions needed
