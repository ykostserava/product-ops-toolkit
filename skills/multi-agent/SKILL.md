# multi-agent

Think in layers:
proposal -> design -> specs -> tasks

## Description

Orchestrate multi-agent collaboration through structured layers of work. Break down complex projects into proposals, designs, specifications, and executable tasks with specialized agents handling each phase.

## Trigger

Use when:
- User asks for multi-agent collaboration or orchestration
- Complex projects need structured breakdown
- Tasks require proposal, design, specs, and implementation phases
- Multiple specialized agents should work together

Keywords: "multi-agent", "layered approach", "proposal to tasks", "agent collaboration"

## Workflow

### Layer 1: Proposal
- Define the problem or opportunity
- Establish high-level goals and constraints
- Identify stakeholders and success criteria
- Create initial scope boundaries

### Layer 2: Design
- Translate proposal into architectural decisions
- Define system components and interactions
- Identify technical approaches and patterns
- Create design artifacts (diagrams, mockups, etc.)

### Layer 3: Specs
- Break design into detailed specifications
- Define interfaces, data models, and contracts
- Establish acceptance criteria
- Document technical requirements

### Layer 4: Tasks
- Convert specs into executable tasks
- Assign tasks to specialized agents
- Coordinate execution and integration
- Track progress and handle dependencies

## Agent Roles

Use the Agent tool with appropriate subagent types:
- **Explore:** research and discovery in proposal / design phases
- **Task-specific agents:** execute implementation tasks

## Best Practices

- Maintain clear handoffs between layers
- Document decisions and rationale at each layer
- Use parallel agent execution for independent tasks
- Consolidate learnings back to earlier layers when needed
- Keep the user informed at layer transitions
